# Per-route Concurrency Limiters & Queuing

Wyre includes a built-in route-level concurrency limiting and request queuing middleware. This is designed to manage bursty, parallel tool calls generated by AI agents without dropping connections or overloading backend systems.

## Why Route-Level Concurrency is Critical for Agents

Unlike human traffic which is spread out, AI agents operate in burst patterns. A single planning loop might execute 10 parallel tool calls in a fraction of a second.

If a server lacks rate/concurrency controls:
1. All tool-calls hitting resource-heavy routes (e.g., calling vector databases, external search, or downstream LLMs) run concurrently.
2. The server's CPU/RAM saturates, causing downstream timeouts.

If using traditional rate-limiters:
1. The server immediately drops requests, causing agent errors.

**Wyre's Concurrency Limiter** offers a balance:
- **Max Concurrent:** It limits the number of active requests executing on a specific route.
- **Queuing:** Excess requests are held in a FIFO queue.
- **Backpressure & Recovery:**
  - If the queue is saturated, it rejects requests immediately with `429 Too Many Requests` (including a `Retry-After` header).
  - If a request sits in the queue longer than a specified timeout, it is rejected with `503 Service Unavailable`.
  - If the client disconnects while waiting in the queue, the request is cleanly discarded from the queue to free up slot resources.

---

## Core API

The middleware is defined in [concurrency.go](file:///c:/projects/oun/concurrency.go):

```go
func ConcurrencyLimiter(
    maxConcurrent int, 
    queueLimit int, 
    queueTimeout time.Duration,
) Middleware
```

## Usage Example

The following code illustrates how to apply the concurrency limiter to a specific route:

```go
package main

import (
	"time"
	"wyre"
)

func main() {
	router := wyre.NewRouter()

	// Slow tool-calling route: Limit to 2 concurrent executions, queue up to 10 requests, 
	// timeout requests waiting in queue longer than 5 seconds.
	slowToolLimiter := wyre.ConcurrencyLimiter(2, 10, 5 * time.Second)

	router.Handle("POST", "/tools/heavy-search", slowToolLimiter(wyre.HandlerFunc(searchHandler)))

	// Other routes remain unaffected by this route's limiter
	router.HandleFunc("GET", "/status", statusHandler)

	server := wyre.NewWithConfig(wyre.DefaultConfig("127.0.0.1:8080"))
	server.ListenAndServe()
}

func searchHandler(w *wyre.ResponseWriter, r *wyre.Request) {
	// Simulated slow DB/vector search
	time.Sleep(1 * time.Second)
	w.WriteFixedBody(200, "text/plain", []byte("Search Results"))
}

func statusHandler(w *wyre.ResponseWriter, r *wyre.Request) {
	w.WriteFixedBody(200, "text/plain", []byte("OK"))
}
```

# Per-Route Concurrency Limiters & Queuing

Wyre includes a built-in route-level concurrency limiting and request queuing middleware. This is designed to manage bursty, parallel tool calls generated by AI agents without dropping connections or overloading backend databases or downstream services.

## Why Route-Level Concurrency is Critical for AI Apps

Unlike human-driven traffic which is distributed, AI agents run in loops that make burst requests. A single planning step might trigger 10 parallel tool queries to a server at the exact same millisecond.

If a server lacks rate/concurrency controls:
1. All tool calls (heavy searches, database queries, downstream LLM calls) execute at once.
2. The server's CPU/RAM spikes, leading to timeouts.

If using traditional IP rate-limiters:
1. The server immediately drops requests, causing agent errors and planning failures.

**Wyre's Concurrency Limiter** provides a production-grade balance:
- **Active Execution Cap**: Limits how many requests can run concurrently on a specific route.
- **Request FIFO Queuing**: Holds overflow requests in a queue instead of dropping them.
- **Disconnect Cancellation**: If a client disconnects while waiting in the queue, the request is cleanly discarded to immediately free up queue slots.
- **Sane Backpressure**:
  - If the queue fills up, new requests are rejected immediately with `429 Too Many Requests` (including a `Retry-After` header).
  - If a request sits in the queue longer than a specified timeout, it is cancelled with `503 Service Unavailable`.

## The Limiter API

Initialize a new limiter middleware with:

```go
func ConcurrencyLimiter(
    maxConcurrent int, 
    queueLimit int, 
    queueTimeout time.Duration,
) Middleware
```

## Usage Example

The following example demonstrates how to wrap a heavy search endpoint with a route-level limiter:

```go
package main

import (
    "time"
    "wyre"
)

func main() {
    router := wyre.NewRouter()

    // Configure a limiter for heavy vector search:
    // - Limit to 2 concurrent active executions
    // - Queue up to 10 requests if active slots are full
    // - Cancel requests waiting in queue longer than 5 seconds
    heavySearchLimiter := wyre.ConcurrencyLimiter(2, 10, 5 * time.Second)

    // Apply only to the resource-intensive route
    router.Handle("POST", "/tools/heavy-search", heavySearchLimiter(wyre.HandlerFunc(searchHandler)))

    // Other lightweight routes remain unaffected by this limiter
    router.HandleFunc("GET", "/status", statusHandler)

    server := wyre.NewWithConfig(wyre.DefaultConfig("127.0.0.1:8080"))
    server.ListenAndServe()
}

func searchHandler(w *wyre.ResponseWriter, r *wyre.Request) {
    // Simulated slow vector/DB search
    time.Sleep(1 * time.Second)
    w.WriteFixedBody(200, "text/plain", []byte("Search complete!"))
}

func statusHandler(w *wyre.ResponseWriter, r *wyre.Request) {
    w.WriteFixedBody(200, "text/plain", []byte("OK"))
}
```
